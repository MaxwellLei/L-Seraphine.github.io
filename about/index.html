关于短期想法

在考研的过程中，我是十分折磨且困扰的，以至于那个时候就想出来很多有待开发的功能软件，但是Win或者Android没有平替或者说已有的软件不能满足现在的要求，短期想法就是维护一个更加偏向于“日记/手账本/日记事”类型的软件自己的开源项目。

2023年了，不会有人还写日记吧，让我看看是谁，原来是我啊🤡

我参考过很多软件，有的是 UI 过于老旧，有的是功能不全面，甚至大部分都是硬排版，不支持Markdown语法，跨平台同步问题，是否支持自定义跨平台，是否支持开源日历协议等等，目前唯一比较接近设想都是一些网页端的软件，然后他们的定位偏向于会议笔记等，加上他们是深度绑定自己的服务器，这也就意味着你的数据没有办法备份导出（如果他们倒闭了），而且你还要购买他们的会员，还不可以支持自己后续编程增加自定义需求等等，而我发现在Mac端却有很多类似的软件，看来Win系统需要我来出手啦😀

到目前我写这篇为止，唯一和我设想的支持：

  1. 自定义服务器跨平台同步
  2. 免费开源
  3. 主打“日记，手账本”功能
  4. 支持Markdown语法
  5. 完全本地，安全，且支持加密算法
  6. 比较现代化的 UI ，且支持自定义样式
  7. 拥有插件来拓展功能

比较满足上述的就只有eDiary和Obsidian，eDiary作者最近发布了他对4.0的更新说明要支持跨平台同步和Markdown，我更加偏向于eDiary作为平替，而不是建议使用Obsidian，Obsidian属于是都想做都做了，但是综合体验缺一般，属于是哪个方面都没做好的“中等”，eDiary我个人认为最大的缺点就是不满足我列举的 6 , 7 两条，所以我决定开发一个全新的软件来填补这个地方的缺口😁

目前项目进度：新建文件夹（笑），还在 UI 设计和软件架构规划中

希望我自己不会鸽掉这个项目😶‍🌫️(笑)，项目日志随记开发日志，最终目前夭折，因为技术选择问题，不得不长期规划。

关于博客框架的建议

如果你也希望搭建一个自己的博客网站，或许你可以参考一下我接下来要说明的一些建议，博客框架类型目前就两种，一种是带有完整前端和后端的架构，常见的网站大部分都是，代表：WordPress，Typecho ，Halo等，另一种是纯静态页面，代表Hexo，Hugo。

对于新手来说，我更加推荐前者，它可以让你更加方便的掌控整个博客网站，缺点就是需要一个活动的服务器。而后者则是在本地编译后仅仅将前端放到了网站上，它意味着更加快速的访问，但是会缺失后端管理的方便和一些功能，当然你也可以使用相关插件来实现和前者类似的功能，就是比较折腾。

如果你希望使用带有完整前端和后端的架构，我推荐使用WordPress，它是个老牌的博客框架了，这就意味着它有着大量的解决方案和插件，主题等，更多的用户意味着更长的更新和寿命，也就意味着更加稳定。当然，最近出来一个现代化的博客框架就是Halo，也就是本站所使用的，它使用的JAVA开发的，也就意味着一定程度上会占用更多的内存，因为它是一个现代化的框架，所以它的 UI 会看起来比较现代化，也就是好看，不过 WordPress 也是存在很多现代化的 UI 主题的，只是相比Halo有着很多新生的东西。

如果你希望使用纯静态页面，我建议使用Hugo，它是采用go写的，这意味着它的编译速度是非常快速的，相比于Hexo，快很多，虽然Hexo官方宣传文章很多的情况下编译速度也就几分钟，但是当我几十篇文章，20多万字的时候，我的机械硬盘编译器来就已经好几分钟了，这样当你写的博客很多的时候，意味着你要花费大量的时间浪费在编译上了，不过Hexo除了编译耗时一点，其他都还是很不错的。如果你不介意每次更新博客的时候，需要等待几分钟的时间可以选择使用。

虽然可能没说的很详细，但是也就当参考吧，就这样祝看到这里小伙伴未来可期🎆
